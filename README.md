## Project 1: Document Upload Governance Workflow

### Overview

A Power Automate workflow designed to improve visibility and governance of project documentation uploaded to SharePoint.

### Business Problem

Project teams were uploading files without a consistent mechanism for notifications, audit tracking or communication. This resulted in manual effort and reduced visibility across stakeholders.

### Solution

The workflow:

1. Detects when a new file is uploaded.
2. Retrieves document information.
3. Sends email notifications.
4. Posts updates to Microsoft Teams.
5. Logs document activity into an Excel tracker.

### Skills Demonstrated

- SharePoint Integration
- Teams Notifications
- Email Automation
- Activity Tracking
- Governance Controls

## Project 2: Automated Project Folder Creation Workflow

### Overview

A scheduled Power Automate workflow developed to automate project folder creation and document management processes.

### Business Problem

Creating project folder structures and copying standard documentation manually was repetitive, time-consuming and susceptible to inconsistency.

### Solution

The workflow:

1. Runs automatically on a schedule.
2. Retrieves source files from SharePoint.
3. Creates new project folders.
4. Copies documentation into the new structure.
5. Processes multiple files using automated looping logic.

### Skills Demonstrated

- Scheduled Automation
- SharePoint Administration
- Loop Processing
- Document Control
- Process Optimisation
